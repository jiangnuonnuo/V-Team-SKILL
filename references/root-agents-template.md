# 项目级 Codex Agent 约束

## 身份门禁

1. 开始任何计划或开发前，必须从用户指令或当前任务上下文确认自己的 `agent-id`。
2. 如果无法确定 `agent-id` 或个人 `AGENT.md` 缺失，必须先读取 `Plan/onboarding.md` 完成快速入职；入职完成前禁止实现代码。
3. 快速入职只在 `Plan/onboarding.md` 指定的范围无法映射或与现有 Agent 明显争夺同一业务模块时询问用户，禁止自行冒充已有身份。
4. 确认身份后，必须先读取 `Plan/agents/<agent-id>/AGENT.md`，再执行其中规定的最小读取顺序。

## 开发门禁

1. 只维护自己的 `Plan/agents/<agent-id>/PLAN.md`，任务项必须是完整功能或明确功能修复。
2. `PLAN.md` 的 review 未通过 `check-plan` 或未经用户批准时禁止实现代码。
3. 按风险选择最小有效验证：定向验证用于单模块、低风险改动；相关完整回归用于跨模块、公共接口、安全或数据迁移；全仓回归只用于发布、全局基础设施或全局配置变更。同一代码、依赖和运行配置未变化时，可复用成功测试证据，不重复执行相同验证。外部真实接口验证必须限频，失败后先分析原因，不进行无条件自动重试；任何再次真实调用都必须在计划中记录触发条件、次数上限和预期证据，无法判断时询问用户。所选验证测试通过后才能进入本地提交流程。
4. Review 检查需求、验收标准、改动范围与风险、测试等级、命令和结果证据；Reviewer 默认不重跑已有且未过期的测试，仅在证据缺失或过期、结果冲突或高风险影响时补充验证。
5. 本地提交前只统一检查一次 `git diff --name-only HEAD`，并按照个人白名单处理越界路径。
6. 白名单外路径必须先向用户说明原因和影响；用户可以只授权当前提交，该授权不得永久扩大白名单。
7. 使用中文完成本地 Git 提交，禁止自行推送远程，禁止自行合并代码。
8. 本约束不创建、切换、命名或管理 Git 分支。

## 上下文规则

1. 默认忽略 `Plan/archive/`。
2. 默认忽略状态为 `completed` 或 `abandoned` 的计划和对接历史。
3. 只读取个人规则指定且与当前任务直接相关的协作文档。
4. 架构、接口或长期依赖发生变化时更新当前态协作文档；简单跨模块修改不强制新增对接文件。
