# 项目级 Codex Agent 约束

## 身份门禁

1. 开始任何计划或开发前，必须从用户指令或当前任务上下文确认自己的 `agent-id`。
2. 如果无法确定 `agent-id`，必须停止开发并询问用户，禁止自行选择或冒充其他身份。
3. 确认身份后，必须先读取 `Plan/agents/<agent-id>/AGENT.md`，再执行其中规定的最小读取顺序。
4. 个人 `AGENT.md` 缺失时禁止继续，先请用户注册或重新生成该身份。

## 开发门禁

1. 只维护自己的 `Plan/agents/<agent-id>/PLAN.md`，任务项必须是完整功能或明确功能修复。
2. `PLAN.md` 未经用户批准时禁止实现代码。
3. 每个任务项完成实现后必须运行足以证明功能正确的测试；测试通过后才能进入本地提交流程。
4. 本地提交前只统一检查一次 `git diff --name-only HEAD`，并按照个人白名单处理越界路径。
5. 白名单外路径必须先向用户说明原因和影响；用户可以只授权当前提交，该授权不得永久扩大白名单。
6. 使用中文完成本地 Git 提交，禁止自行推送远程，禁止自行合并代码。
7. 本约束不创建、切换、命名或管理 Git 分支。

## 上下文规则

1. 默认忽略 `Plan/archive/`。
2. 默认忽略状态为 `completed` 或 `abandoned` 的计划和对接历史。
3. 只读取个人规则指定且与当前任务直接相关的协作文档。
4. 架构、接口或长期依赖发生变化时更新当前态协作文档；简单跨模块修改不强制新增对接文件。
